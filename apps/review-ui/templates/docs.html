PolicyCapture Documentation

Local-first screen recording, frame extraction, OCR, and entity recognition pipeline for policy documentation.

PolicyCapture records screen sessions, intelligently samples and scores frames, runs OCR with entity extraction, and produces structured, searchable data from recorded policy workflows. The entire system runs locally at localhost:8420 with zero cloud dependencies.

Processing Pipeline

The pipeline transforms a raw screen recording into scored, classified, OCR-processed frames in 8 stages.

1. Frame Sampling

Extracts frames from video at configurable intervals using timestamp-based sampling (critical for WebM files with unreliable FPS metadata).

Uses CAP_PROP_POS_MSEC instead of frame-counting to handle variable frame rates. Default interval: 0.5 seconds. 
# sample_frames.py
while True:
    timestamp = cap.get(cv2.CAP_PROP_POS_MSEC)
    if timestamp >= next_sample_ms:
        # Capture this frame
        cv2.imwrite(path, frame)
        next_sample_ms += interval_sec * 1000

2. Quality Metrics

Blur Score (Sharpness)

Blur Score Formula
blur_score = min(laplacian_variance / 5000.0, 1.0)
Uses Laplacian variance to detect edges/detail. Higher = sharper. Threshold for "sharp": ≥ 0.15

Stability Score

Stability Score Formula
MSE = mean((frame1 − frame2)²)
stability_score = 1.0 − (MSE / 255.0²)
Compares consecutive frames. 1.0 = identical, 0.0 = completely different. Default for first frame: 0.5

3. Scene Change Detection

Compares each frame against the last kept frame (not the immediately previous one) using a dual metric approach.

Scene Change Detection
combined_similarity = 0.65 × SSIM + 0.35 × hist_correlation
is_scene_change = combined_similarity < 0.92
SSIM (Structural Similarity Index) captures perceptual quality changes. Histogram correlation catches color distribution shifts. Both computed on 320×180 grayscale.
SSIM 65%
Histogram 35%

Visual Importance

After scene change detection, each frame gets a composite visual importance score:

Visual Importance Formula
visual_importance = 0.40 × scene_change_score
         + 0.20 × text_density
         + 0.25 × relevance_score
         + 0.15 × blur_score
Scene 40%
Text 20%
Relevance 25%
Blur 15%

4. Relevance Scoring

Combines keyword matching from OCR text with visual structure analysis (line detection for tables/forms).

Relevance Score (with text)
keyword_score = min(matched_count / (total_keywords × 0.3), 1.0)
structure_score = 0.6 × min(h_lines/10, 1) + 0.4 × min(v_lines/5, 1)

relevance_score = 0.7 × keyword_score + 0.3 × structure_score
Without OCR text, falls back to pure structure_score. Structure detection uses Canny edges + Hough line transform.

Default Keywords

Matched against OCR text (case-insensitive substring search):

demographics income household members eligibility application policy address amount case number benefits enrollment coverage deductible premium copay provider plan medicaid medicare snap tanf wic

5. Candidate Scoring

The final composite score that determines which frames get auto-selected.

Candidate Score Formula
candidate_score = 0.35 × blur_score
        + 0.45 × relevance_score
        + 0.20 × stability_score
Relevance 45%
Blur 35%
Stability 20%
Relevance — policy keyword matches + visual structure
Blur — image sharpness (Laplacian variance)
Stability — frame-to-frame consistency

Best frame selection groups frames into 4-second windows and picks the highest-scoring candidate from each window.

6. Redundancy Detection

Two-Stage Deduplication
Perceptual Hash8×8 image hash
threshold: 0.92
→
SSIM ComparisonFull image SSIM
threshold: 0.95
→
Mark Redundantcandidate_score = 0

Stage 1 (Deduplication): Perceptual image hash (8×8 grayscale → 64-bit binary hash). Hamming distance similarity ≥ 0.92 = duplicate. Higher-scored frame kept.

Stage 2 (Redundancy): SSIM comparison of scene-change frames. Similarity > 0.95 = redundant. Frame's candidate_score set to 0.0 and flagged.

OCR Engine

Enhanced Tesseract OCR with image preprocessing and structured text extraction that preserves document layout.

OCR Preprocessing Pipeline
Input Image
→
Upscale<1200px → 2×
→
DenoisefastNlMeans
→
CLAHEContrast enhance
→
SharpenKernel filter
→
TesseractPSM 3, eng

Preprocessing Details

  • Upscaling: Images < 1200px wide are scaled 2× with INTER_CUBIC for better glyph recognition
  • Denoising: fastNlMeansDenoising(h=10, templateWindowSize=7, searchWindowSize=21)
  • CLAHE: Contrast Limited Adaptive Histogram Equalization (clipLimit=2.0, tileGridSize=8×8) for uneven lighting
  • Sharpening: 3×3 kernel convolution with center weight 9

Structured Text Extraction

Uses Tesseract TSV output with block_num, par_num, line_num fields to reconstruct document layout with proper line breaks and paragraph spacing. Confidence threshold: ≥ 20 for partial, ≥ 30 for full words.

Table Detection & Extraction

Table Extraction Pipeline
Canny Edge Detectionthreshold: 50, 150
↓
Detect H/V LinesMorphological operations
↓
Find Grid RegionsMin area: 0.5% of image
↓
Extract Cell BoundariesLine projection analysis
↓
OCR Each CellPSM 7 (single line mode)
↓
Build Structured Gridrows × cols with text

Table cells are identified by detecting horizontal and vertical line intersections. Each cell is OCR'd independently in single-line mode (PSM 7) for maximum accuracy. Falls back to block-level OCR (PSM 6) if cell detection fails.
